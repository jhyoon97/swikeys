# 스위치 리서치 Skill

사용자가 특정 제조사의 스위치를 리서치해달라고 요청하면 이 절차를 따릅니다.

## 절차

1. **웹 리서치**: 해당 제조사의 스위치 정보를 웹에서 수집합니다.
   - 공식 사이트, 리뷰 블로그, 키보드 커뮤니티 등을 참고합니다.
   - 가능한 모든 스위치 모델을 빠짐없이 수집합니다.

2. **결과 정리**: `docs/{제조사명}-switches-research.md` 파일에 아래 포맷으로 정리합니다.

3. **표 컬럼**: `src/types/switch.ts`의 `KeyboardSwitch` 인터페이스를 기준으로 하되, 아래 컬럼을 사용합니다:

| 컬럼명 | 설명 | 비고 |
|--------|------|------|
| 이름 | 스위치 전체 이름 (영어) | |
| 한글이름 | 스위치 한글 이름 (제조사 포함) | 공식 한글명이 있는 경우만 기재, 없으면 `-` |
| 제조사 | 스위치를 **생산하는 공장/회사** | 콜라보 파트너 제외 |
| 콜라보업체 | 디자인/기획 파트너 | 없으면 `-` |
| 스위치타입 | `리니어`, `택타일`, `hall effect` 중 택1 | |
| 저소음 | `Yes` / `No` | 사일런트 스위치 여부 |
| 상부하우징재질 | | |
| 하부하우징재질 | | |
| 스템재질 | | |
| 공장윤활 | `Yes` / `No` / `Partial` | |
| 스프링길이(mm) | | 미확인 시 `-` |
| 마운트핀 | `5` / `3` | |
| 트래블(mm) | 총 트래블 거리 | |
| 입력지점(mm) | 액추에이션 포인트 | |
| 입력압(g) | 액추에이션 포스 | |
| 초기압(g) | 이니셜 포스 | |
| 바닥압(g) | 바텀아웃 포스 | |
| 출처 | 정보 출처 URL (plain URL) | 마크다운 링크 아님 |

4. **표 분류**: 스위치 타입별로 섹션을 나누어 정리합니다:
   - 리니어 스위치
   - 택타일 스위치
   - 사일런트 스위치는 별도 섹션 없이 `저소음` 컬럼에서 `Yes`로 구분

5. **제조사 vs 콜라보업체 구분 규칙**:
   - **제조사**: 스위치를 물리적으로 생산하는 회사 (예: HMX, Gateron, Cherry 등)
   - **콜라보업체**: 디자인, 기획, 커스텀 사양을 담당한 파트너 (예: Sillyworks, Kinetic Labs 등)
   - 이름에 `HMX x Partner` 형태로 표기된 경우 `Partner`가 콜라보업체
   - 콜라보업체가 여러 곳이면 쉼표로 구분 (예: `SPM, Soulcat`)

6. **데이터 규칙**:
   - 확인할 수 없는 값은 `-`로 표기
   - 무게 변형(Light/Heavy 등)은 별도 행으로 분리
   - 출처는 plain URL로 기재 (마크다운 링크 문법 사용하지 않음)
   - **한글이름**: 공식적으로 사용되는 한글 명칭이 있을 때만 기재. 영어 이름을 억지로 한글로 음역하지 않음. `HMX V0T` 같이 한글 표현이 없는 경우 `-`로 표기. **반드시 제조사명을 포함**하여 검색 시 혼동을 방지 (예: `HMX 프로그`, `게이트론 옐로우`, `체리 흑축`). 제조사 한글명이 통용되면 한글로, 아니면 영문 그대로 사용
